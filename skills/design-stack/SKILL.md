# Design Stack

Offer design capability without silently installing or outsourcing anything.

## Procedure

1. Identify the design job: concept, wireframe, visual direction, component spec, or asset.
2. Decide whether existing repo design patterns are binding.
3. Offer tools or agents with cost, install command, data exposure, and fallback.
4. If approved, create a design charter and first task spec.
5. If rejected, continue with repo-native HTML/CSS/SVG or written brief.

## Offer Shape

```text
Need:
Option:
Install/access:
Cost/risk:
Fallback:
First task:
```

External tools, uploads, paid services, and public sharing require explicit approval.
