# Consilium

Collect independent opinions before synthesis.

## Procedure

1. Frame the decision, constraints, and success criteria.
2. Give each reviewer the same prompt and hide other opinions.
3. Ask for recommendation, reasoning, risks, and confidence.
4. Compare disagreements, not prose quality.
5. Synthesize one decision with evidence and a fallback.

## Roles

Use 2-4 perspectives: product, engineering, risk, operations, domain expert, or contrarian. Route by model class, not model name.

## Output

Return: decision, why, rejected alternatives, confidence, risks, and next verification step.
