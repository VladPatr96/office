# Generic Blueprint

Use when no domain-specific blueprint fits.

## Starting Departments

| Department | Owns | First task |
|---|---|---|
| Board | goals, issues, roadmap | define first measurable outcome |
| Engineering | implementation artifacts | create first shippable slice |
| QA | verification | define release gate |
| Docs | public/internal docs | make install/use path clear |

## Intake Questions

- What is the first user-visible outcome?
- What must not change?
- What is the fastest evidence that the project is moving?
- What action requires approval?

## Initial KRs

- KR1: first verified slice shipped.
- KR2: onboarding path works from README.
- KR3: one confirmed lesson captured.
