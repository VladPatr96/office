# done-check

Используй перед закрытием issue, публикацией результата или статусом "готово".

Проверь: outcome виден, DoD evidence есть, fresh verifier не вернул unresolved FAIL, docs обновлены при изменении поведения/установки/структуры, нет лишних файлов, риски зафиксированы.

Вердикты: `DONE`, `NEEDS_WORK`, `BLOCKED`, `UNVERIFIED`. На `UNVERIFIED` не закрывать.
