# Frame

Turn ambiguity into a decision-ready problem statement.

## Procedure

1. Separate user goal, proposed solution, constraints, and success criteria.
2. Identify what can be inferred from repo/docs/current evidence.
3. Mark true unknowns and whether each is blocking.
4. Recommend a default for non-blocking choices.
5. Produce a one-paragraph frame and the next skill to use.

## Output

Use this shape:

```text
Problem:
Success:
Constraints:
Assumptions:
Blocking questions:
Recommended next step:
```

If there are more than three blocking questions, switch to `grill-me`.
